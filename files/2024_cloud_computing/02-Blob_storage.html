Cloud Computing on Azure - Assignment 3: Working with Azure Blob Storage

Objective:

This assignment focuses on Azure Blob Storage and demonstrates how to integrate a cloud-based application with Blob storage through a virtual machine (VM). Students will learn how to create and manage Blob storage, upload and retrieve data programmatically, and securely connect a VM to Blob storage using Azure SDKs.

Part 1: Setting Up Azure Blob Storage using the Portal

  1. Task: Create a new Azure Blob Storage Account and configure it for storing unstructured data like files, images, and documents.

  2. Instructions:

    • In the Azure portal, navigate to Storage Accounts and click Create.
    • Set the following:
      • Resource Group: Use the existing resource group (CloudBasics_RG).
      • Storage Account Name: Choose a unique name (e.g., myblobstorage01).
      • Region: Same as your other resources.
      • Performance: Choose Standard.
      • Replication: Choose Locally Redundant Storage (LRS).
    • Once the account is created, navigate to the Containers section and create a new container:
      • Container Name: Use a unique name (e.g., myfiles).
      • Public Access Level: Set to Private (no anonymous access).
    • Once the container is created, create a file using touch file.txt, fill it with whatever you prefer and upload it to the blob storage via the portal.

Part 2: Uploading and Managing Files in Blob Storage using the CLI

  1. Task: Upload files to Azure Blob Storage using the Azure portal and Azure CLI.

  2. Instructions:

    • In the Azure portal, upload at least two sample files (e.g., text files, images) into the container (myfiles) you created earlier.
    • Next, use the following commands to upload another file: 
      az storage blob upload \
  --account-name <your-storage-account-name> \
  --container-name <your-container-name> \
  --name <blob-name> \
  --file <local-file-path> \
  --auth-mode login
    • Verify the upload by listing the contents of the Blob container using: 
      az storage blob list \
  --account-name <your-storage-account-name> \
  --container-name <your-container-name> \
  --output table
    • Download the file you uploaded to the storage in Part 1 using the CLI.
    • Copy the previous container to a new one using the CLI. All contents shoould be copied.
